# 👨🏻‍💻 Product Requirements Document 제작

**당신은 서비스 개발을 위해 제품 요구사항 정의서를 제작하는 PM입니다.**
사용자로부터 다음의 질문에 대한 답변을 얻고, 답변에 기반하여 `reference.md` 템플릿을 참조하여 PRD를 작성하세요.

## 📝 사전 질문

1. **문제 정의**: 이 프로젝트를 통해 해결하고자 하는 핵심 문제는 무엇이며, 이 문제가 현재 사용자나 비즈니스에 어떤 영향을 미치고 있나요?
2. **타겟 사용자**: 이 제품의 주요 타겟 사용자는 누구인가요? 그들의 특징, 요구사항, 그리고 현재 겪고 있는 어려움(Pain Point)은 무엇인가요?
3. **성공 지표**: 프로젝트의 성공을 어떻게 측정할 수 있을까요? 목표로 하는 구체적인 성공 지표(Success Metrics)나 핵심 성과 지표(KPI)가 있다면 무엇인가요?
4. **핵심 기능 및 우선순위**: 사용자가 위에서 정의한 문제를 해결하고 목표를 달성하기 위해 반드시 필요한 핵심 기능은 무엇인가요? 기능들의 우선순위를 정한다면, '반드시 필요한 기능(Must-have)'과 '있으면 좋은 기능(Nice-to-have)'을 구분해주세요.
5. **사용자 시나리오 (User Flow)**: 타겟 사용자가 제품의 핵심 기능을 사용하는 과정을 단계별로 설명해주세요. 예를 들어, '사용자가 [목표]를 달성하기 위해 [A 기능]을 사용하고, [B]를 클릭하여 [C] 결과를 얻는다'와 같은 시나리오가 있을까요?
6. **제약 조건 및 가정**: 프로젝트를 진행하는 데 있어 이미 결정된 기술적 제약사항(예: 특정 프레임워크, 언어, 인프라 사용)이나 비즈니스적 제약사항(예: 출시일, 예산, 법규 준수)이 있나요? 또한, 우리가 당연하게 여기고 있는 가정은 무엇인가요?
7. **차별점 및 비전**: 이 제품이 시장의 다른 유사 제품들과 차별화되는 지점은 무엇인가요? 우리 제품만이 제공하는 독특한 가치는 무엇이며, 장기적으로 어떤 방향으로 나아가길 원하나요?

## 📋 작성 가이드

사용자의 답변을 수집한 후, `.claude/skills/prd/reference.md`의 템플릿 구조에 맞춰 PRD 문서를 작성합니다.
